# 案例图片处理规范

## 目的 1

本文件用于约束本目录下 EURUSD 案例图片的统一处理方式。

后续切换到新的 session 时，可直接让模型先读取本文件，再继续执行图片处理任务。

适用范围：

- `E:\EURUSD案例库\案例概览图`
- 目录下所有 `EURxx` 案例图片，不限制只到 `EUR21`
- Excel 数据源：`交易机会统计(已自动还原).xlsx`

---

## 核心原则

1. 图片字段内容以 Excel 为唯一权威来源。
2. 图片编号以 Excel 最后一列“编号”为准，例如 `EUR01`、`EUR02`。
3. 图片分类目录以“交易机会类型 + 时间级别”决定。
4. 对于新截图，只追加底部字段信息栏，不做复杂擦除。
5. 对于旧图修复，只有在明确要求时才尝试擦除旧字段。
6. 若图片已经损毁或擦除困难，优先让用户重新截图，不强行修图。
7. 文件名保持编号格式，例如 `EUR08.png`。

---

## 数据来源规则

字段必须从 Excel 中读取，不凭图片原有文字判断。

读取 Excel 字段值时，默认需要先去掉首尾空格，再参与编号匹配、分类映射和文字渲染。

使用的 Excel 文件：

- `E:\EURUSD案例库\案例概览图\交易机会统计(已自动还原).xlsx`

使用的字段顺序固定为：

1. 年限
2. 品种
3. 交易机会类型
4. 时间级别
5. 机会释放复杂度
6. 积累内部结构复杂度
7. 释放形式(正向/逆向)
8. 边线是否共振
9. 释放复杂度
10. 是否到达目标位
11. 最大盈利倍数
12. 加仓位置
13. 加仓释放复杂度
14. 加仓最大盈利倍数
15. 数据来源
16. 相关因子
17. 目标位之后是否延续释放
18. 编号

---

## 文件命名规则

统一命名为：

- `EUR01.png`
- `EUR02.png`
- ...
- `EUR21.png`

不保留原始日期命名，不保留 `Snipaste_...` 之类临时文件名。

如果用户把新截图放进 `未处理` 文件夹，且文件名已经是编号名，例如 `EUR08.png`，则直接处理。

---

## 分类归档规则

处理完成后，图片要移动到对应目录。

目录映射如下：

- `上通道 + H1` -> `通道\上通道\H1`
- `上通道 + H4` -> `通道\上通道\H4`
- `下通道 + H1` -> `通道\下通道\H1`
- `下通道 + H4` -> `通道\下通道\H4`
- `双底 + H1` -> `双头双底\双底\H1`
- `双底 + H4` -> `双头双底\双底\H4`
- `双头 + H1` -> `双头双底\双头\H1`
- `双头 + H4` -> `双头双底\双头\H4`
- `复合头肩顶 + H1` -> `头肩形态\复合头肩顶\H1`
- `复合头肩顶 + H4` -> `头肩形态\复合头肩顶\H4`
- `头肩顶 + H4` -> `头肩形态\头肩顶\H4`
- `头肩顶 + H1` -> `头肩形态\头肩顶\H1`
- `头肩底 + H4` -> `头肩形态\头肩底\H4`
- `头肩底 + H1` -> `头肩形态\头肩底\H1`
- `上旗 + H1` -> `旗形\上旗\H1`
- `上旗 + H4` -> `旗形\上旗\H4`
- `上旗子 + H1` -> `旗形\上旗\H1`
- `上旗子 + H4` -> `旗形\上旗\H4`
- `下旗 + H1` -> `旗形\下旗\H1`
- `下旗 + H4` -> `旗形\下旗\H4`
- `上三 + H1` -> `三角形\上三\H1`
- `上三 + H4` -> `三角形\上三\H4`
- `下三 + H1` -> `三角形\下三\H1`
- `下三 + H4` -> `三角形\下三\H4`

如果 Excel 中出现了未写进本节的新组合，处理前先检查磁盘上是否已有对应目录。

- 如果已有目录，按现有目录归档，并同步把新映射补回本文档。
- 如果没有目录，不要凭空猜测归档位置；应先和用户确认，或按现有大类结构新建后再把规则补进本文档。

---

## 新截图处理规则

当用户重新截图后：

1. 不做顶部旧字段擦除。
2. 不尝试修改图中行情、线条、标注。
3. 直接在图片底部追加信息栏。
4. 信息栏背景色取图片主背景色。
5. 信息栏内容从 Excel 当前最新数据读取。
6. 信息栏追加完成后，按编号命名。
7. 最后按分类规则移动到对应文件夹。

---

## 旧图修复规则

当用户要求“把旧图改成完整格式”时：

1. 可以清除顶部旧字段区域。
2. 回填色必须和图片主背景色一致。
3. 不允许出现黑色、紫色或明显不一致的填充块。
4. 如果修复后图片损毁明显，应停止继续修复，改为让用户重新截图。

若图片出现以下情况，视为不适合继续修复：

- 原始顶部文字残留严重
- 顶部出现黑条或异常色带
- 底部原字段无法干净清除
- 清除后破坏行情主体

这类图片应移动到 `未处理`，等待用户重新截图。

---

## 底部信息栏样式规范

### 总体布局

- 只在图片底部追加信息栏
- 不覆盖原行情图
- 分成两行字段
- 每行 9 列
- 总共 18 个字段

### 字体层级

标题字体：

- 字号：`15`
- 字重：常规
- 颜色：`#9C6A5A`

数据字体：

- 字号：`23`
- 字重：加粗
- 颜色：`#D94841`

### 分隔线

- 信息栏顶部使用浅色分隔线
- 颜色应接近：`#E8DCC2`

### 背景色

- 使用图片主体的米白背景色
- 不允许纯黑、深灰、紫色等异常填充

---

## 文字内容规则

1. 标题使用 Excel 第一行字段名。
2. 数据使用对应编号所在行的字段值。
3. 字段顺序不能变。
4. 过长字段允许自动换行。
5. 若内容过长仍放不下，可以在末尾截断并加 `...`。

---

## OCR 使用规则

OCR 只用于辅助检查，不作为最终数据来源。

可使用 OCR 的场景：

- 判断某张未编号图片可能对应哪个编号
- 校验图片内部文字是否和 Excel 大致一致
- 排查手动改名是否错误

不可使用 OCR 的场景：

- 直接用 OCR 结果替代 Excel 字段
- 在 Excel 明确存在数据时，凭 OCR 覆盖 Excel

---

## 未处理文件夹规则

`未处理` 文件夹用于存放：

- 用户新截图的原图
- 已判定损毁、等待替换的图片
- 临时待归类的图片

处理流程：

1. 用户把新截图放入 `未处理`
2. 若文件名已是编号名，则直接读取
3. 按 Excel 追加底部字段
4. 移回对应分类目录
5. `未处理` 中不保留处理后的副本

补充约定：

- `未处理` 中优先处理文件名已经是 `EURxx.png` 的图片。
- 如果图片处理完成后已移动到分类目录，则 `未处理` 应为空，或只保留仍需人工确认的图片。

---

## 操作边界

允许做的事：

- 读取 Excel
- 给图片追加底部字段栏
- 按编号重命名
- 按类型和级别归类
- 将损毁图片移到 `未处理`

不允许默认做的事：

- 无备份情况下强行大范围擦除原图
- 覆盖用户新截图内容
- 凭猜测修改 Excel
- 找不到原图时声称可以恢复历史版本

---

## 出现问题时的处理方式

### 情况 1：图片损毁严重

处理方式：

- 直接停止修复
- 移到 `未处理`
- 让用户重新截图

### 情况 2：没有备份

处理方式：

- 明确告知无法恢复最初原图
- 不伪造恢复结果
- 让用户提供新截图

### 情况 3：图片内部文字和 Excel 不一致

处理方式：

- 以 Excel 为准
- 如用户要求，可先做 OCR 校验说明差异

### 情况 4：命令行里中文显示乱码

处理方式：

- 读取 Markdown、PowerShell 输出文本时，先设置控制台编码为 UTF-8。
- 推荐使用：`[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`
- 不要因为终端显示乱码就判断文件本身损坏，先确认只是控制台编码问题。

### 情况 5：Excel 无法读取或被占用

处理方式：

- 优先使用正式文件 `交易机会统计(已自动还原).xlsx`，忽略以 `~$` 开头的临时锁文件。
- 如果存在 `~$交易机会统计(已自动还原).xlsx`，说明 Excel 可能正开着，但只要正式文件能读，就继续处理。
- 如果正式文件被独占且无法读取，再提示用户关闭 Excel 后重试。

### 情况 6：Excel 字段包含首尾空格

处理方式：

- 先对读取到的字段值执行首尾空格清理，再做分类和命名判断。
- 不要因为例如 `双头 ` 这样的尾部空格，就误判成“没有目录映射”。

## 建议的后续工作方式

后续新 session 中，模型应按以下顺序工作：

1. 先读取本文件
2. 再读取 `交易机会统计(已自动还原).xlsx`
3. 检查 `未处理` 中是否有新截图
4. 给新截图追加底部字段栏
5. 按编号放回对应分类目录
6. 如果用户要求校验，再做 OCR 对照

为了加快处理，当前目录可直接使用：

- `py -3 process_case_images.py`

该脚本默认行为：

- 读取当前目录下第一个非 `~$` 开头的 `.xlsx`
- 只处理 `未处理` 文件夹内的 `EUR*.png`
- 读取 Excel 字段后追加底部信息栏
- 按“交易机会类型 + 时间级别”自动归档
- 处理成功后删除 `未处理` 中对应原图

如果后续修改了目录映射或字段顺序，应同步更新：

- `案例图片处理规范.md`
- `process_case_images.py`
